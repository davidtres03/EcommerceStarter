# ?? Auto-Update System

## Overview

The EcommerceStarter installer includes **automatic update detection** that keeps both the installer and the application up-to-date from GitHub releases!

---

## ? Features

### **Installer Auto-Update**
- ? Checks GitHub on every launch
- ? Compares current version with latest release
- ? One-click download & apply
- ? Automatic restart with new version
- ? Non-intrusive (can skip update)

### **Application Auto-Upgrade**
- ? Detects latest EcommerceStarter release
- ? Downloads from GitHub
- ? Applies updates safely
- ? Preserves all data
- ? Runs database migrations

---

## ?? How It Works

### **On Installer Launch:**

```
1. Installer starts
2. Check admin privileges
3. Check GitHub API for latest release
4. Compare versions
5. If newer:
   ?? Show update dialog
   ?? User clicks "Yes"
   ?? Download new installer
   ?? Launch new installer
   ?? Exit old installer
6. Proceed with install/upgrade
```

### **On Application Upgrade:**

```
User clicks "Upgrade"
    ?
Check GitHub for latest app release
    ?
Download EcommerceStarter-vX.X.X.zip
    ?
Stop IIS
    ?
Backup files
    ?
Extract new files
    ?
Run migrations
    ?
Restart IIS
    ?
Done!
```

## ?? GitHub Release Structure

### **Required Files in Each Release:**

1. **Installer** (optional - for installer updates):
   - `EcommerceStarter-Installer-v1.0.0.exe`

2. **Application** (required - for app updates):
   - `EcommerceStarter-v1.0.0.zip`
   - Contains published application code

3. **Migrations** (optional - if schema changes):
   - `migrations-v1.0.0.sql`
   - Or handled via EF migrations in code

4. **Release Notes** (auto-generated by GitHub):
   - Shows what changed
   - Displayed to user before update

### **Example Release:**

```
Tag: v1.1.0
Name: Version 1.1.0 - Bug Fixes & Improvements

Assets:
?? EcommerceStarter-Installer-v1.1.0.exe (2.5 MB)
?? EcommerceStarter-v1.1.0.zip (15 MB)

Release Notes:
## What's New
- Fixed admin login CreatedAt issue
- Added upgrade detection
- Improved installer UI
- Security patch for XSS

## Database Changes
- Added CreatedAt column to AspNetUsers

## Breaking Changes
None
```

## ??? Technical Implementation

### **UpdateService.cs**

```csharp
// Check for installer updates
var updateInfo = await updateService.CheckForUpdatesAsync();

// Check for application updates  
var appUpdate = await updateService.CheckForApplicationUpdatesAsync();

// Download and apply installer update
await updateService.DownloadAndApplyInstallerUpdateAsync(url);

// Download application update
var zipPath = await updateService.DownloadApplicationUpdateAsync(url);
```

### **GitHub API Integration**

- **Endpoint:** `https://api.github.com/repos/{owner}/{repo}/releases/latest`
- **Rate Limit:** 60 requests/hour (unauthenticated)
- **Response:** JSON with release info, assets, notes

## ?? Workflow

### **Development & Release:**

1. **Make changes** in code
2. **Test locally**
3. **Commit to Git**
4. **Create GitHub Release:**
   ```bash
   git tag v1.1.0
   git push origin v1.1.0
   ```
5. **Upload assets** to release:
   - Application zip
   - (Optional) New installer
6. **Write release notes**
7. **Publish release**

### **User Update:**

1. **User runs installer**
2. **Sees "Update Available" dialog**
3. **Clicks "Yes"**
4. **Installer auto-updates**
5. **Restarts with new version**
6. **User proceeds** with fresh install or upgrade
