## Technical Details

### Production Environment Configuration

```
ASPNETCORE_ENVIRONMENT = Production
ASPNETCORE_HTTPS_PORT = (empty for HTTP/reverse proxy)
ASPNETCORE_DETAILEDEERRORS = false
```

### Generated Configuration Files

**appsettings.Production.json**
```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning",
      "Microsoft.EntityFrameworkCore": "Warning"
    }
  },
  "DetailedErrors": false,
  "IncludeExceptionDetails": false
}
```

**web.config** (Generated by installer)
- Security headers (CSP, X-Frame-Options, etc.)
- HTTP/2 compression for JSON, JS, CSS
- Static content caching strategy
- Request filtering and size limits
- HSTS support for HTTPS environments
- ReadyToRun enabled app pool

### Performance Optimizations

| Optimization | Impact | Result |
|---|---|---|
| Release Build | Removes debug symbols | 50% smaller deployment |

| Tiered Compilation | Optimized execution | Better long-term perf |
| HTTP/2 Compression | Dynamic content compression | 50-80% bandwidth savings |

## Troubleshooting

### Issue: Build fails "dotnet CLI not found"
**Solution:** Install .NET 8 SDK from https://dotnet.microsoft.com/download

### Issue: Application shows Development mode
**Solution:** This won't happen! Release build is enforced. If it occurs:
1. Check web.config for ASPNETCORE_ENVIRONMENT = Production
2. Restart IIS app pool
3. Clear browser cache

### Issue: Installer won't run
**Solution:** 
1. Run as Administrator (right-click ? Run as administrator)
2. Ensure Windows Installer is enabled
3. Check Event Log for error details

### Issue: Slow startup time
**Solution:**
1. First request is normal (ReadyToRun optimization kicks in)
2. Subsequent requests will be faster
3. Enable app pool preload for consistent performance
